## Overview

Effect Resolution defines how game events are processed and how players interact with those events.

Every game event is resolved deterministically.

At any point in time, there is only one active game event being resolved.

Unless explicitly stated otherwise, no game event may interrupt another game event currently being resolved.

---

# Game Events

A Game Event is any action that changes the game state.

Examples include:

- Playing a card
    
- Moving a chess piece
    
- Capturing a piece
    
- Promoting a pawn
    
- Attaching a Unit
    
- A Unit leaving play
    
- Drawing a card
    
- Discarding a card
    
- A triggered ability activating
    
- A card effect resolving
    
- A player gaining mana
    
- A player losing mana
    

Game Events are processed immediately according to the rules in this section.

---

# Event Queue

Game Events are resolved using a First-In, First-Out (FIFO) queue.

When an action generates one or more Game Events:

1. The generated events are added to the end of the Event Queue.
    
2. Events are resolved one at a time.
    
3. Resolving an event may generate additional Game Events.
    
4. Newly generated events are added to the end of the queue.
    
5. Resolution continues until the Event Queue is empty.
    

The game may not advance to the next phase while the Event Queue contains unresolved events.

---

# Triggered Abilities

Triggered abilities activate automatically when their trigger condition occurs.

Triggered abilities do not resolve immediately.

Instead, they generate a new Game Event and are placed into the Event Queue.

Example triggers include:

- When this piece moves...
    
- When this piece captures...
    
- When this card is played...
    
- At the beginning of your turn...
    
- At the end of your turn...
    

---

# Priority

Priority determines which player may perform optional actions.

Outside of the Reaction Phase:

- The active player has priority.
    
- No Reaction cards may be played unless explicitly permitted.
    

During the Reaction Phase:

1. The active player receives priority.
    
2. The active player may play one legal Reaction card or activate one legal Reaction ability.
    
3. Priority passes to the opposing player.
    
4. The opposing player may play one legal Reaction card or activate one legal Reaction ability.
    
5. Priority alternates between players until both players pass consecutively.
    
6. Once both players pass consecutively, all pending Game Events are resolved.
    

Players may continue responding until both players pass without performing an action.

---

# Reactions

Reaction cards may only be played during the Reaction Phase unless explicitly stated otherwise.

Every Reaction card must define the event or condition that allows it to be played.

Examples include:

- After a piece moves
    
- After a piece is captured
    
- When a Unit is attached
    
- When a Spell is played
    
- When your King is placed in check
    

If the triggering condition no longer exists when the Reaction would be played, it cannot be played.

---

# Effect Resolution

When resolving a Game Event:

1. Verify the event is still legal.
    
2. Apply the event's effects.
    
3. Generate any resulting Game Events.
    
4. Place generated Game Events into the Event Queue.
    
5. Continue resolving the Event Queue.
    

Effects always resolve completely before the next event begins resolving.

---

# Illegal Effects

If an effect becomes illegal before it resolves, only the illegal portion of the effect fails.

Any remaining legal portions continue resolving unless the card explicitly states otherwise.

Examples include:

- A targeted piece leaves the board before the effect resolves.
    
- A target no longer satisfies the card's targeting requirements.
    
- A card effect attempts to move a piece that can no longer legally move.
    

---

# Simultaneous Events

Multiple Game Events may be generated by a single action.

When this occurs:

1. The active player determines the order of their own simultaneously generated events.
    
2. The opposing player determines the order of their simultaneously generated events.
    
3. The active player's events are added to the Event Queue first.
    
4. The opposing player's events are added immediately after.
    

Resolution then proceeds normally.

---

# Replacement Effects

Some card effects replace a Game Event rather than triggering after it.

Replacement effects modify the event before it occurs.

Example:

"Instead of being captured, return this piece to its starting square."

Replacement effects are applied before the original Game Event resolves.

If multiple replacement effects apply, the active player chooses the order in which they are applied.

---

# Continuous Effects

Some effects remain active after resolving.

Continuous effects remain active until:

- Their duration expires.
    
- Their source leaves play.
    
- Another card removes them.
    
- The game ends.
    

Continuous effects are continuously considered when determining the current game state.

---

# Last Known Information

If an effect references information about an object that has left play, the game uses that object's final known public game state immediately before it left play.

---

# Cannot Takes Precedence

If one effect states that an action can occur and another states that it cannot occur, the prohibition takes precedence.

Example:

"This piece cannot be targeted by Spells."

Overrides:

"Target any piece."

---

# Resolution Completion

A Game Event is considered resolved only after:

- Its effects have completed.
    
- All Game Events generated by it have been added to the Event Queue.
    

Once the Event Queue is empty, play proceeds according to the current phase of the turn.
